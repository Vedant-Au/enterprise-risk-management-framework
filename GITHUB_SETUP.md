# GitHub setup

## Create and upload

1. On GitHub, select **New repository**.
2. Name it `enterprise-risk-management-framework`, set it to **Public**, and do not add starter files.
3. Select **uploading an existing file**.
4. In Finder, open the extracted folder and press **Command + Shift + .** so `.github` and `.gitignore` appear.
5. Drag everything inside the folder into GitHub. Do not drag the ZIP or outer folder.
6. Check for `.github/workflows/quality.yml`, `.gitignore`, `README.md`, `analysis.py`, `data`, `docs`, `outputs` and `tests`.
7. Commit directly to `main` with `Add enterprise risk management analytics framework`.

## Verify and present

1. Confirm the README renders the prioritisation chart and risk heat map.
2. Open **Actions** and confirm the `quality` workflow passes.
3. Add description `Risk register, FMEA and governance roadmap for a multi-site engineering SME.`
4. Add topics: `enterprise-risk-management`, `risk-analysis`, `python`, `fmea`, `iso-31000`, `coso`, `governance`.
5. Pin for risk, consulting, operations or governance applications.
